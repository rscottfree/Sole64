BEGINNING TUTORIAL

Writing Your First Program

Once you start SOLE 64, you can begin programming right away. READY?

1) Type new and hit enter.* This will clear the memory so we can start a new program.

2) Type 1 print('What did you do!?') and hit enter. This enters some code at line 1.

3) And now let's run our program! Type run and hit enter.

You told the computer to print some text, so you should see that text printed out when you run the program. Now you will want to save your creation.

*The new command will erase whatever program you have in memory, so if you have work in memory you do not want to lose, make sure you save it before you run this command.

Saving Your Program

To save your program, type save myprogram and hit enter.

That's it. You should get a confirmation that your file saved. To make sure the save really worked and to load your program later, continue reading in the next section.

File names can only have letters, numbers and underscores in them.

Loading Your Program

Loading your program is as easy as saving it. Let's start out by clearing the memory.

Type new and hit enter.

Now when you type run nothing will happen because there is no program in memory.

Okay, now load the program you have saved.

Type load myprogram and hit enter.

You should see a message that says your program was loaded. Now let's run it.

Type run and hit enter.

The program you wrote before should now run.

Summary

In this tutorial you have learned how to use the print function to print text to the screen. Functions are the building blocks of programs. In the following sections you will learn how to use the built-in functions as well as create functions of your own.

Working with Programs

In this section and subsequent sections, examples will be shown and quick explanations given. These tutorials are not meant to be comprehensive, but to give you a grasp on the tools you can use to write your own programs. So make sure you try to understand what the code is going; learn to read the code.

Some Vocabulary

Variable: There are two types of variables, strings and numbers. Variables are like things your program can use.

Function: Functions are tasks for the computer to perform. You can tell functions to use parameters to do stuff. For example, you can have a cut function and a paper variable, and you can pass the paper variable to your cut function as a parameter. And then your cut function can do someting with your paper variable, like cut it or something.

Blow your mind: You can pass variables to functions and you can also pass function to functions.

The Program

The building blocks of programs written on SOLE 64 are the functions print and input, which allow you, the programmer, to display text to the user and get user input.

Type in this program and run it.

Remember, you can use the new command to clear memory in order to start a new program. To see what is in memory, use the list command.

10 print('What is your name?')
20 DEF getName(name)
30 print('Hello, ' + name)
40 END getName
50 input(getName)

This program asks the user what their name is, gets input from the user, then says hello to the user using the name the user entered. To do this, the print and input functions are used, as well as a user-created function called getName.

Notice that because line 30 is within the getName function, it is not run immediately when you run the program. Instead, that line will only be run when the getName function is called. And getName is not called until the user enteres something to satisfy the input function.

User defined functions begin with DEF and end with END. In this example, we create a function called getName with one parameter called name. All this function does is print out the name that was passed in.

This program also uses the input function. This is a built-in function that asks for user input. Notice that the input function has one parameter passed to it, and in this case we are passing our user defined function, getName.

Any parameter you pass to the input function must be a function. And this function must have at least one parameter. That parameter will be the text the user entered.


Add a Function

Add the following lines to your program.

60 DEF printSomeStuff(my_string, my_number)
70 print(my_string)
80 print(my_number)
90 END printSomeStuff
35 printSomeStuff('a string here', 34)

Run the list command and your program should look something like this:

10 print('What is your name?')
20 DEF getName(name)
30 print('Hello, ' + name)
35 printSomeStuff('a string here', 34)
40 END getName
50 input(getName)
60 DEF printSomeStuff(my_string, my_number)
70 print(my_string)
80 print(my_number)
90 END printSomeStuff

Here we have added a new function called printSomeStuff, and that function takes two parmaeters, my_string and my_number. We then call that function within the getName function, and pass it a string and a number. Our new function simply prints out that string and number that we pass it.

If you need to clean up the line numbering in your program, or you're running out of lines, you can use the renumber command. To renumber your program and separate each line of code by 5 lines, you would run renumber 5.

You can also insert lines anywhere in your program with the insert command. To insert 3 lines at line 30, you would run insert 3 at 30. If you have code on lines 30 and above, they will be shifted up 3 lines to make room for your inserted lines.

copy

If you're looking how to copy text on a line instead of lines in a program, click on this link to see the section on cut/copy/paste.

copy 5-10 to 20 Copy lines 5 through 10 to line 20

Use the copy command to copy lines of code to other locations within your program or to a copy buffer. When you perform the copy command, the selected lines are inserted in at the line you specify. If you specify a copy buffer, the lines of code are copied to this buffer for you to get later when you need them. You can access the copy buffers with paste command and the listb command. Copy buffers are places in memory where you can store code. If you copy code to a buffer that already has code in it, the old code will be overwritten by the new code.

If you have downloaded a file to device 7 (as a file to your computer), then to load it you need to drag the file that you downloaded onto SOLE 64. The save command saves your program. You can use the load command to load it again. The save command takes your lines of code that are in memory and saves them to be loaded later. You can specify a device or not; if you do not specify a device, device 8 will be used. Device 8 is the default device, which is your local computer. Device 9 is the remote storage device. Device 7 is your local disk--or in other words, saving to device 7 will generate a file and download it to your computer. This file will probably be called "download". In order to load this file again, you can drag and drop it onto SOLE 64. (Device 7 is experimental.) Programs uploaded to remote storage are public, so specifying an author allows multiple people to have the same program name. Also, when you save to remote storage (device 9), you will be prompted to enter a password. If you have saved this same filename to remote storage before, you must use the password you used the first time; otherwise, you may enter whatever password you want. (But don't forget your password!) Uploading to remote storage requires an internet connection.

SUMMARY OF PROGRAMMING COMMANDS

delay functionality

@2000:print('hi') Print 'hi' after 2000 milliseconds (2 seconds)
@3500:paintCircle(100, 100, 30) Draw a circle after three and a half seconds (3500 milliseconds)

The delay ability allows you to run some code after a certain number of milliseconds. You can print something, call a function, draw a circle--whatever you want as long as it is a single command. To make this work, start the line with @ then enter a number of milliseconds for the delay, then : and then the code you want to run.

The delay functionality is asynchronous. This means that any lines of code after the line with the delay function will be executed right away; other code will not wait for the code passed to the delay function to finish. Take this program, for example:

10 print('one') // this will print first
20 @2000: print('two') // this will print third
30 print('three') // this will print second

The output of this program would be:

one
three
two

clear

clear() Clear the screen

The clear command is used to clear the screen of text. This is often useful if you want your program to start out with a clean screen.

get

get('myname', processMyName) Loads the value labeled 'myname' and passes that value to the user-defined function processMyName.

The get function is for loading data you have saved using the put function. (See more about the put function below.) The get function is asynchronous, meaning you need to provide a callback function to receive the result of this function. (See more about the asynchronous function in the note below, next to the put function description.) You can store any variable you want, and when you get it with the get command, it will come back exactly as you saved it. In the example here, whatever was saved as 'myname' will be passed to the user-defined processMyName function. That function might look like this:
10 DEF processMyName(my_name)
20 print(my_name)
30 END processMyName

include

include(myProgramName, callback) Load a local program and call the callback
include(myProgramName) Load a local program without a callback

With this command you can include another program within the current program. Maybe you have a program that has some useful functions in it--you can include this program within many other programs so you don't have to copy and paste lots of code or have duplicate code in different programs. You can include a callback function that will be called once the included program has finished loading. If you need to make sure that the program has fully loaded (like before you go calling functions from the included program) you should use this callback function.

input

input(myFunction)

The input command is used to get user input. The input command takes one parameter, a function. This is generally a user-defined function, but it could also be any of the print functions, although that is less helpful.

Any user-defined function that is passed to input must itself accept one parameter. This parameter will be the text the user inputed. So you can create a function that accepts some text (the user input) and you can do something with that input.

poke

poke('bg', 'blue') Change background color to blue
poke('fg', '#000000') Change the foreground color to black
poke('border', '#ffffff') Change the border to white
poke('blink', 200) Set the cursor blink rate to 200 milliseconds
poke('tts', true) Turn on text-to-speech
poke('tts_enqueue', true) Queue the text-to-speech instead of interrupting it
poke('tts_rate', 1.5) Set the speech rate to 1.5

This command helps the user change certain system properties not otherwise easily accessible.

When specifying colors, you can use CSS color names or Hexadecimal colors (sometimes called a Hex color). For a list of CSS color names, see this list on w3schools.com.

Hex colors start with a # and are typically followed by 6 digits. The first two digits represent the amount of RED color present, the third and fourth represent the amount of GREEN color, and the last two digits represent the amount of BLUE color. And these amounts are in hex format, meaning the lowest amount is 0 (zero) and the highest is f. E.g. on one end you have #000000 (black) and on the other side of the spectrum you have #ffffff (white). For more information about hex colors, see this w3schools page about them.

prin

l prin('hi') Prints the text 'hi' to the screen
prin(10) Prints the number 10 to the screen

The prin command is like the print command except that it does not create a new line after it is done printing your text to the screen This means if you use the prin command, the cursor will end up at the end of the text you printed, instead of on a new line below the text you printed. This command will not overwrite the current line, but append to the end of the current line. Be careful though, using the print command after this command will erase anything you printed (because print does overwrite the current line).

print

print('hello there') Prints the text 'hello there'
print(10 + 2) Prints the number 12 to the screen
print(my_name) Prints the value of the variable my_name to the screen. So if the value of my_name were 'Roger', the text 'Roger' would be printed to the screen
print('Hello, my name is ' + my_name) If the value of my_name is 'Roger', the text 'Hello, my name is Roger' would be printed to the screen
print("Also okay") Prints the text 'Also okay' to the screen.

The print command prints text to the screen. You can pass the print command a string, a number, a variable, or an expression, like 31+2 or combination of string and variable. After each print command, a new line is created. Also, you can use single quotes or double quotes to specify text within the print command.

printc

printc('I am centered') Prints the text 'I am centered' in the center of the screen.

This is just like the print command except it will print to the center of the screen, meaning all printed text will be center aligned.

printr

printr('Hey, over here on the right!') Prints out the desired text on the right of the screen, or right aligned

Just like the print command except it right aligns the given text.

put

put('mysavedata', my_name, doneSaving) Saves the variable my_name to the computer under the name 'mysavedata', and calls the user function doneSaving, when the computer is done saving the data.
put('mysavedata', my_name) Saves the variable like the first example, but does not call a user function.
put('mysavedata', 'a new string') Saves the given string under the name 'mysavedata'.

It may be useful to save some information within your program. The put command saves a user-defined variable to the computer, giving it a name so it can be loaded again later. This is an asynchronous function, so you can give it a callback to be notified when the computer is done saving. Also see the get command.

An asynchronous function is one that does not wait for its tasks to finish before telling your program to move on to the next function or command. For example:

10 put('mynametosave', my_name)
20 print('hi there!')

The put command is asynchronous. This program will save something to the computer and then print something to the screen. Because put does not wait for things to finish before letting the program move on, the line that prints 'hi there!' to the screen may run before my_name is actually saved.

If you need to make sure that the next line runs only after you are done saving, you need to pass a callback function. A callback function is a function you define that is called when an asynchronous function is done processing its tasks. In this example, adding a callback function might look like this:

10 put('mynametosave', my_name, doneSavingName)
20 print('hi there!')
30 DEF doneSavingName()
40 print('done saving the name')
50 END doneSavingName

random

random(1, 10)

The random command will return a number between the two numbers that you give it. So random(1, 10) will return a whole number between 1 and 10 inclusive, meaning 1 and 10 are also possibilities.

range

range(1, 10) Return list with the numbers 1 through 10
range(1, 10, 2) Return list with the numbers 1 through 10, going by 2s, e.g. [1, 3, 5, 7, 9]

Range is handy for quickly getting a list of numbers.

ADVANCED TOPICS

Cut/Copy/Paste

There are two different ways to cut or copy text. We'll start with the keyboard version:

To copy some text, hold down the SHIFT key and then use the right and left arrow keys to select the text you want to cut or copy. Once you have highlighted the text you want to copy, you can hold down the CONTROL (CTRL) key and tap the C key. This will copy the text. If instead you want to cut the text (remove it and save it so you can paste it somewhere else later), you would hold down the same CONTROL key and then tap the X key. If you do this, you will see that the text you selected is removed.

If you want to paste the text you just copied or cut, you hold down the CONTROL key and tap the V key. This will paste the text wherever you have your cursor.

If you have some text selected, but decide you don't want to copy or cut it, just let up on the SHIFT key and hit any other key and the highlight will disappear.

The second option for copying text is with the mouse. Simply hold down the CONTROL key and click the line of text you want to copy. This will copy the whole line, and make it ready for you to paste it. You use the same keys to paste as you did before, holding down the CONTROL key and tapping the V key. This will paste it where your cursor is located.

If you don't want to copy the whole line, you will have to use the keyboard option of copying text, positioning your cursor with the arrow keys, holding down SHIFT while you select the text you want.

Using Tab to Get Stuff Done

In SOLE 64 you can use the tab character at certain times in order to auto-complete commands. This section will show you how.

Tab for History

When you run the history command, you will notice that by each command in your history there is a number. You can use this number to then run the command. Say the number of the command in history is 5. To run that command without typing the whole thing in again, type this:

!5 and then hit the tab key.

You should notice that the command associated with the number 5 will replace the text you have entered. Now you can hit enter to run that command.

If you do not want to look up the number for a command in history, but you know what the command started with, you can auto-complete by using the first characters of the command.

Type !his and then hit the tab key. The last command you ran that began with 'his' will replace the text you have entered. You can then hit enter to run that command

Tab Line Numbers

If you enter the number of a line of code in your program and hit tab, the line is completed with the line from your program. If there is no line of code entered for that line number, the cursor simply advances a space, ready for you to start coding. This is useful if you need to edit a line of code.

Up and Down Arrow Your History

You can use the up and down arrows on your keyboard to scroll through history. The up arrow moves you back in time in your history, and the down arrow moves you forward in time. Once the history command you want is under your cursor, simply hit enter to run that command.

Delete Like You Mean It

Deleting one character at a time is okay, I guess, but when you really need to ax some words quickly, you can use the CTRL (control) key and the BACKSPACE or DELETE key to delete whole words at a time. Holding down the CTRL key while tapping the BACKSPACE or DELETE key will delete one word at a time.

Or maybe you want to delete a whole line at once? Just hold down the CTRL key and tap the 'd' key, and the entire line you are on will be deleted.

Move by the Word

Similar to the ability to delete whole words at a time, you can also navigate a line whole words at a time. Hold down the CTRL (control) key and use your right or left arrow keys to move one word to the right or left.

Quick Save

If you find yourself saving frequently (and you should!), you might get tired of typing in the save command or finding it in your history. There is a shortcut: Ctrl-s. Hold down the CONTROL (Ctrl) key and tap the 's' key, and the last save command you typed in will appear. Then all you have to do is hit enter.

PROGRAMMING BASICS

This section will go through some of the basic topics in programming. In general, it will show you about variables, how to store text and numbers, how to manipulate text and numbers, how to store arrays of text and numbers, how to loop through your arrays, and how to make conditional statements. It will also explain about functions and objects, which is to say you will learn how to define tasks you want the computer to perform, and you will learn how to define objects that your functions can work with.

Summary of Functions and Objects

In SOLE 64, functions are your verbs, the action phrases to get stuff done. A function might be called goNorth, or eatFood. You can name your functions whatever you want. The normal style for function names is to use camel case, where the first letter of each word is capitalized except for the first word. A camel case function name in SOLE 64 might look like this: eatSomeFoodThenTakeANap.

Functions also control the flow of your program. They take the user from one task to another task. Once one function finishes, it generally calls another function. Perhaps you have a program that adds one to every number the user enters. You would have a function that got user input, that would call another function that added one to the number the user inputed, print that new number out, then call the function again that asks for user input. That would be an example of a simple flow from one function to another.

More on Functions

As noted, functions are tasks you can ask the computer to perform; they are blocks of code that do something specific or general--it is up to you. They can take parameters and they can return parameters. The basic syntax of a function might look something like this:

10 DEF eat(what, with_what)
20 print('I am eating ' + what + ' with a ' + withWhat)
30 return 'all done'
40 END eat

This code defines a function called eat. Eat takes two parameters, what and with_what. Inside the function, line 20 prints out some text, including the parameters that were passed in. Line 30 returns a string. The last line, line 40, ends the function.

Variables

Variables are the basic things you can pass around your program and functions, and your functions can in turn do things with those things, those variables. You can declare a new variable like this:

var my_new_variable = 'hi'

This creates a new variable and sets it's value to a string, 'hi'. If you do not set the value of a variable, it will automatically get the value of undefined. Undefined is a special value given to a variable that is not defined. Like this:

var my_new_variable

When variables are passed into functions, you can use them within that function and then return them if you like. Variables are great for passing around information.

Arrays

Arrays are for storing related variables. You can create an array of any number of variables. An array of things to eat might look like this:

var my_array = ['pie', 'oranges', 'honey', 'vegetable oil']

Some things you can do with an array:

print(my_array[0]) Print the first item in your array. (Arrays item numbering always starts with 0.)

print(my_array.length) Print the lengh of your array.

my_array[3] = 'something' Set the 4th item (remember it is numbered starting with 0) to the string 'something'.

my_array.push(22) Add the value 22 to the end of the array.

print(my_array.pop()) Remove and print the last item in the array.

You can find a lot of great info and more ways to use arrays through this link. This will take you to the Mozilla Developer Network where there is great documentation on everything JavaScript.

Objects

Objects are like dictionaries. They are made of of words, and definitions for those words. Or in other words, they are groups of keys and values, where one key points to one value. For example:

var my_object = { 'my key': 'my value', 'another': 20 }

Notice that the keys and values can be either strings or numbers. You access the values within your object like this:

print(my_object['my key']) will print out 'my value'

print(my_object['another']) will print out 20

You can set keys and their values like this:

my_object['third key'] = 'my third value'

You can also push objects onto arrays just like any other variable.

Objects are the building blocks of programming; use them well. Any information you need to store can be stored in an object, and if you need to keep some objects together, either put them in another object (the value of a key can be an object) or in an array.

Comments

You can use comments to leave notes for yourself in the code. Comments have no effect on how your code runs. You can use them like this:

print('some stuff') // My comments here
print('more') /* another comment */

When you use // to comment, everything after the // is commented. If you use /* */, everything between the /* and the */ is commented.

if Statement

If statements are for when you want something to happen only if a certain condition is met.

if (your_name == 'Biff') {
  print('Hi Biff')
}

This will only print 'Hi Biff' if the variable your_name is equal to 'Biff'. Note the use of the double equals (==). When assigning variables, you use the single equals. When comparing them to see if they are equal, use the double equals. You can also use != which means not equal to. Or in the case of numbers you can use < and > to mean less than or greater than, or <= and >=, which means less than or equal to and greater than or equal to.

else Statement

You can use the else statement along with the if statement. So if something is true, do one thing, else (otherwise) do something else.

if (your_name == 'Biff') {
  print('Hi Biff')
} else {
  print('You are not Biff')
}

This can also be written like this:

if (your_name == 'Biff') print('Hi Biff')
else print('You are not Biff')

So if the condition in the if statement is not true, it will run the code that is in the else statement.

Loops

Remember arrays? Arrays are lists of variables. Sometimes it's really useful to be able to loop through an array. Say you have an array of fruits and you want to print out each fruit name. You can do that pretty easy with loops. Here is an example.

var fruits = ['Apple', 'Pear', 'Peach', 'Kumquat']
for(var fruit in fruits) print(fruit)

The for means you're starting a loop (often called a for loop), and inside you say var fruit in fruits which means go through your list of fruits and for every item, assign it to the temporary variable fruit and print that variable out, then move on to the next item in the array of fruits.

If you have to do more things within your loop, you can use the curly braces like you do for the if statement. Like this:

var fruits = ['Apple', 'Pear', 'Peach', 'Kumquat']
for(var fruit in fruits) {
   prin('I like ')
   print(fruit)
}

PAINT GRAPHICS

SOLE 64 provides some basic painting functions you can use to draw shapes on the screen.

setPaintColor

setPaintColor('white') Set the drawing color to white.

The setPaintColor command is used to change the color that is used to draw whatever you are going to draw. If you want to draw a white circle you first have to set the color to white. If you don't set a color, the color black will be used.

paintCircle

paintCircle(100, 100, 20) Draw a circle at location (100, 100), with a diameter of 20.
paintCircle(100, 100, 20, true) Draw a filled circle.

The paint circle commands draws circles! The first two numbers in the function make up the location you want to draw your circle at; this is the center of your circle. The first number is the x coordinate (the number of pixels from the left side of the screen) and the second number is the y coordinate (the number of pixels from the top of the screen). The third number is the diameter of the circle (the distance from one side of the circle to the opposite side of the circle). If you set the last parameter to true, it will draw a filled circle instead of just the outline.

paintOval

paintOval(100, 100, 30, 50) Draw an oval at location (100, 100), with a width of 30 and a height of 50.
paintOval(100, 100, 30, 50, true) Draw a filled oval.

The paint oval command draws an oval (a.k.a. ellipse), with a center give in the first two parmeters, and a width and height given in the second two parameters. The last parameter can be set to true in order to draw a filled oval.

paintLine

paintLine(100, 50, 200, 100) Draw a line from (100, 50) to (200, 100)

Draws a simple line with the specified width. The first two numbers are the starting point of the line, and the next two numbers are the end point of the line.

paintAngleLine

paintAngleLine(100, 100, 20, 55) Draw a line with a center at (100, 100), 20 pixels long, at an angle of 55 degrees

Draw lines at a certain angle. The first two numbers is the location (the center of the line), the third number is the length of the line. The fourth is the angle of the line (which is a number from 0 to 360).

About angles: An angle of 0 is a flat, horizontal line. An angle of 90 is a straigh up and down line. You can think of angles as the points on the outside of a circle. You start at 0 and move around the circle until you come to 360, which is back to the beginning (meaning 360=0). If you set an angle of 360, it would look the same as setting an angle of 0. Or an angle of 390 would look the same as an angle of 30. Go ahead, try it out.

paintPath

paintPath([[100, 50], [150, 100], [200, 50]], 2) Draw a path following the points given.

Draw a series of lines, or a path, with the given points to follow. The first parameter in the function is a list of lists. This specifies the locations. Each location is a list with two numbers, the x and the y coordinates. In the above example, we give three points, (100, 50), (150, 100), (200, 50). When you connect the dots, you get a path that looks like the letter V.

setPaintWidth

setPaintWidth(5) Set the line width that you paint at to 5.

This sets the line width for painting. Once you set the line width, any graphics calls like paintCircle, paintLine, and paintPath, that come after it will use this line width when they are painted.

3D GRAPHICS

SOLE 64 provides some basic functions for drawing and animating 3D shapes. This functionality is provided by Three.js, a 3D rendering library.

setColor

setColor(0xffffff) Set the drawing color to white.
setColor(0xff0000) Set the color to red
setColor(0x000000) Set color to black

The setColor command is used to change the color that is used to draw whatever you are going to draw. If you want to draw a red cube you first have to set the color to red. If you don't set a color, the color red will be used.

cube

cube(100, 100, 100) Create a cube

This creates a cube object. The first parameter is the cube width, the second the cube height, and the third the cube depth. Note that these numbers do not translate to number of pixels, but generic units within the 3D space. Also, this command, like the other 3D drawing commands, will not immediately draw the object to the screen when it is created. This function simply gives you back a variable that represents the cube object. To display it, run the show command, passing in the newly created cube object as the only parameter, like this:

10 var c = cube(100, 100, 100)
20 show(c)

To position a cube, use its position property, like c.position.x = 100 or c.position.y = 150.

To set the rotation of the cube, use the rotation property, like c.position.rotation.x = 50. Other rotation properties include y and z. The position property also supports the z property.

sphere

sphere(100, 20, 20) Create a sphere object with a radius of 100, 20 segments in the width and 20 segments in the height.

Creates a sphere object in 3D space. The first parameter is the radius, and the next two, the number of segments that run vertically and horizontally. As with the cube command and other 3D commands, you must pass this to the show command to display it to the screen.

cylinder

cylinder(70, 70, 200, 20) Create a cylinder object with a top radius of 70, a bottom radius of 70, a height of 200, and 20 segments.

A cylinder. You can set the bottom or top radius to 0, and the segments to 4 to draw a pyramid.

ANIMATING 3D OBJECTS

In order to animate your 3D objects you need to create a function within your program, and pass it to the animation system in SOLE 64. Here is an example of how you might do that.

10 clearGraphics()
20 var c = cube(100, 100, 100)
30 show(c)
40 DEF update()
50 c.rotation.x += 0.01
60 END update
70 startAnimation(update)

This quick program clears the graphics, draws a cube, and then creates a function called update. This function is passed to the animation system with the startAnimation() function. Now, the animation system will call your update function as much as 60 times per second (slower, depending on the speed of your computer). You can use the stopAnimation() function to stop the animation, or clearGraphics() to clear all the graphics, or use the stop command or the keyboard shortcut for the stop command Control-q, which trys to halt the program that is currently running and clears all graphics.
